# 工作流契约（纯 Markdown 模式）

本契约定义了在纯 Markdown 模式下生成专利交底书时的关键检查点。对用户可见的唯一输出物为最终 Markdown。

## 阶段 1：确认模板来源
输入：
- 用户想法与目标
- 可选用户自定义 Markdown 模板

检查点：
- 已判断是否存在用户模板。
- 若无用户模板，默认采用内置模板。
- 已明确最终采用的 Markdown 模板版本。

## 阶段 2：通过对话收敛写作方向
输入：
- 模板结构
- 用户想法、约束条件与目标

检查点：
- 已收集用户初始想法。
- 已提出 2 到 4 个写作方向，且用户可选择或自定义方向。
- 已围绕所选方向明确总体思路、核心对象和潜在保护重点。
- 对话中已显式呈现关键假设、未决事项和边界条件。
- 对理解门槛较高的关键概念，已补充其定义、职责或上下游关系。
- 用户已明确确认所选方向。

## 阶段 3：形成写作方案并让用户确认
输入：
- 已确认的写作方向
- 模板结构

输出：
- 写作方案（用于评审，不是最终交付物）

检查点：
- 写作方案已写清方案目标、核心对象、输入、输出、核心步骤或结构。
- 关键参数、变量、条件、阈值、系数或公式符号已尽量说明含义、来源和用途。
- 关键步骤具备输入、动作、输出闭环。
- 当前仍需用户确认或补充的内容已被显式列出。
- 用户已对写作方案进行评估、纠正或确认。

## 阶段 4：检查可实施性
输入：
- 已修正的写作方案

检查点：
- 不存在未解释来源的核心参数、阈值、权重、系数、评分或控制量。
- 不存在“只有公式没有参数来源”的情况。
- 关键结构、步骤、条件和效果之间的关系可以被清晰追溯。
- 若仍有关键缺口，继续追问或显式标注，而不是直接成稿。

## 阶段 5：生成最终 Markdown
输入：
- 已确认且通过检查的写作方案
- 最终采用的 Markdown 模板

输出：
- 最终 Markdown（交付物）

检查点：
- 输出为一份完整 Markdown 文档，而不是片段。
- 章节结构与模板主结构保持一致，或在必要时做最小补充。
- 模板中的提示语和占位信息已尽量替换为真实技术内容。
- 文风符合工程师技术交底，而不是正式专利申请文件。
- 专业术语使用准确，但不会因为术语堆砌而影响专利代理人的理解。
- 输出聚焦最终交底书正文，不附带额外过程产物。
